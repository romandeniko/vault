#### Методы
GET --- безопасный, идемпотентный метод для получения информации.
POST --- неидемпотентный небезопасный метод для создания ресурсов.
PUT --- идемпотентный метод для обновления ресурсов (перезапись).
PATCH - идемпотентный метод для обновления ресурсов (изменение).
DELETE - идемпотентный метод для удаления ресурсов.

#### URL

* Имя существительное
* Множественное число
* Версия API
* От большего к меньшему
* Переменные пути {}: обязательный параметр, единственное число, представляет собой идентификатор ресурса для выполнения действия
* Параметры запроса: пары ключ-значение: необязательный параметр, множественное число, используется для фильтрации, сортировки, пагинации, поиска и т.д.

Пример:
```
GET http://host/api/v1/profiles/{profileID}/friends?key1=value1&key2=value2
```
