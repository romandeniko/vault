Basic Auth: включает отправку имени пользователя и пароля перед каждым запросом. Передается в header Authorization в виде строки Base64

OAuth/Bearer Token: как правило используется для передачи данных аутентификации в клиент-серверных приложениях. Пример: JSON Web Token: открытый стандарт для создания токена в формате JSON.

Пользователь вводит логин/пароль, сервер высылает JWT-токен на клиент. Клиент формирует Bearer JWT-токен, который высылает на сервер. Сервер авторизует пользователя.

JWT-токен состоит из трех частей:
* Header: тип токена и алгоритм шифрования
* Payload: данные для авторизации пользователя
* Signature: строка для верификации токена
