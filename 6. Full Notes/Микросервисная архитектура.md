Модель архитектуры, основанная на ряде називисимо разверьываемых модулей.
У каждого микросервиса своя база данных.

#### Компоненты:
* микросервисы
* реестры служб: содержит список всех запущенных микросервисов
* шлюз API
* балагсировщик нагрузки
* централизованная конфигурация
* БД для каждого сервиса
* контейнеризация/оркестрация
* безопасность и кправление идентификацией
* CI/CD

#### Преимущества:
* гетерогенность, возможность построить систему с помощью разных языков программирования и технологий;
* децентрализованное управление данными, каждый микросервис содержит свой набор данных, доступный другим микросервисам через соответствующий интерфейс;
* независимость инфраструктуры, каждый микросервис можно разворачивать и изменять независимо от других;
* масштабируемость, расширяются только те сервисы, которые в этом нуждаются.

#### Недостатки
* высокий порог вхождения
* сложность взаимодействия между сервисами
* транзакционность
* нарушение согласования и принципов DRY
